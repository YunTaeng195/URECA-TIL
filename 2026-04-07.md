🚀 오늘 배운 핵심 (Summary)
자바의 개발 환경 구성(JDK, JRE) 및 에디션별 특징 이해

자바의 실행 메커니즘(컴파일 & 인터프리트)과 하이브리드 방식 학습

컴퓨터 구조(폰 노이만)와 변수 및 데이터 타입의 기초 확립

💡 학습 내용 (Main Concepts)
1. 자바 개발 환경 및 에디션
JRE (Java Runtime Environment): 컴파일된 자바 프로그램을 실행시킬 때 필요.

JDK (Java Development Kit): 자바 프로그램을 개발할 때 필요 (JRE 포함).

Java Editions:

SE (Standard Edition): 기본 문법, 알고리즘 풀이 등 표준 개발 환경.

EE (Enterprise Edition): 서버급, 대규모 기업용(Servlet/JSP 포함). 24시간 운영되는 대규모 시스템에 적합.

ME (Micro Edition): 임베디드용(현재는 거의 사용 안 함).

2. 자바의 실행 방식 (WORA: Write Once Run Anywhere)
컴파일(Compile): 소스 코드(.java)를 기계가 이해하기 쉬운 **바이트코드(.class)**로 변환.

인터프리트(Interpret): 변환된 바이트코드를 JVM 위에서 한 줄씩 실행.

자바의 특징: 컴파일러와 인터프리터를 모두 사용하는 하이브리드 방식. 이 덕분에 운영체제에 상관없이 어디서든 실행 가능함.

3. 컴퓨터 구조와 변수
폰 노이만 구조: 입력 → 저장(Memory) → 출력의 흐름과 **연산(CPU)**이 핵심.

입출력 버퍼: 데이터를 전송할 때 임시로 저장하는 공간(in, out).

변수(Variable): 데이터를 담는 그릇이며, 재활용 가능한 메모리 영역.

4. 데이터 타입 (Data Type)
정수형: byte (1), short (2), int (4 - 기본), long (8)

실수형: float (4), double (8 - 기본)

논리형: boolean (true, false)

표기법: * PascalCase: 클래스명 (첫 글자 대문자)

camelCase: 변수명, 메서드명 (첫 글자 소문자, 다음 단어 대문자)

🛠 주요 포인트 & 궁금증
Unsigned 사용 불가?

자바는 전통적으로 C언어와 달리 unsigned 키워드를 지원하지 않음. 즉, 모든 정수형 데이터 타입이 부호(+, -)를 가짐.

(참고: Java 8부터 Integer.toUnsignedLong 등의 메서드로 처리할 수는 있으나, 타입 자체가 존재하는 것은 아님)

💭 느낀 점 & 내일의 목표
느낀 점: 단순히 코드만 짜는 게 아니라 컴퓨터 구조론(폰 노이만)과 연결해서 공부하니 변수가 왜 메모리 영역인지 더 명확히 이해됨.

내일의 목표: 타입 변환(Casting)과 연산자에 대해 깊이 있게 학습하기.
