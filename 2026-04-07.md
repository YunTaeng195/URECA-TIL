## 💡 학습 내용

### 1. 자바 실행 흐름 (Execution Flow)
자바는 "Write Once, Run Everywhere"를 실현하기 위해 컴파일과 인터프리트 방식을 혼합하여 사용한다.

1.  **소스 파일 작성:** `.java` 파일 생성
2.  **컴파일 (Compiler):** `javac` 명령어가 소스 파일을 **바이트코드(`.class`)**로 변환
3.  **실행 (JVM):** `java` 명령어로 JVM(자바 가상 머신) 구동
4.  **인터프리트 & 하이브리드:** JVM 내의 **인터프리터**와 **JIT 컴파일러**가 바이트코드를 해당 OS가 이해할 수 있는 기계어로 번역하여 실행

### 2. 자바 에디션 분류
* **SE (Standard):** 기본/알고리즘용. 표준 에디션.
* **EE (Enterprise):** 대규모 서버/기업용 (Servlet, JSP 등). 24시간 운영 환경.
* **ME (Micro):** 임베디드용 (현재 거의 미사용).

### 3. 변수와 자료형 (Data Type & Byte)
변수는 데이터를 담는 그릇이며, **재활용 가능한 메모리 영역**을 의미한다.

| 분류 | 타입 | 크기 | 특징 |
| :--- | :--- | :--- | :--- |
| **정수** | `byte` | 1 byte | -128 ~ 127 |
| | `short` | 2 byte | |
| | `char` | 2 byte | 유니코드 문자 (음수 없음) |
| | **`int`** | **4 byte** | **자바 정수 연산의 기본** |
| | `long` | 8 byte | 큰 정수 (접미사 `L`) |
| **실수** | `float` | 4 byte | 접미사 `f` |
| | **`double`**| **8 byte** | **자바 실수 연산의 기본** |
| **논리** | `boolean` | 1 byte | `true`, `false` |

* **참고:** 자바는 기본적으로 `unsigned`(부호 없는 정수)를 지원하지 않음.

### 4. 자바 표기법 (Naming Convention)
자바는 가독성을 위해 엄격한 명명 규칙을 권장한다.

| 분류 | 표기법 | 예시 | 특징 |
| :--- | :--- | :--- | :--- |
| **클래스 (Class)** | **PascalCase** | `MemberService`, `UserRole` | 모든 단어의 첫 글자를 대문자로 시작 |
| **메서드 (Method)** | **camelCase** | `getMemberName()`, `save()` | 첫 단어는 소문자, 두 번째 단어부터 대문자 |
| **변수 (Variable)** | **camelCase** | `userName`, `totalCount` | 메서드와 동일 (첫 글자 소문자) |
| **상수 (Constant)** | **UPPER_SNAKE**| `MAX_STOCK`, `DEFAULT_PAGE` | 모두 대문자, 단어 사이는 언더바(`_`) |
| **패키지 (Package)** | **lowercase** | `com.company.project` | 소문자만 사용, 마침표(`.`)로 계층 구분 |

* **참고 사항:**
    * **클래스명:** 명사형으로 짓는 것이 관례 (예: `Member`, `Order`).
    * **메서드명:** 동사형으로 시작하는 것이 관례 (예: `calculateAmount`, `printResult`).
    * **변수명:** 의미를 알 수 없는 `a`, `b` 보다는 `studentAge`처럼 의미 있는 이름을 권장.

### 5. 실습 코드 분석 (Code Examples)
어제와 오늘 실습한 예제 코드들을 통해 자바의 기초 문법을 확인했다.

#### ① 출력 및 서식 (`PrintTestt`)
* `print()`, `println()`의 차이와 `printf()`를 이용한 서식 출력 학습.
* `%s`(문자열), `%d`(정수), `%f`(실수) 등의 지시자 사용.
* `%10.3f`처럼 출력 폭과 소수점 자리를 지정하는 방법 확인.

#### ② 데이터 입력 (`ScannerTest`)
* `Scanner` 클래스를 사용하여 키보드 입력 처리.
* `nextInt()`, `nextDouble()`은 숫자만 가져오고, `next()`는 공백 전까지, `nextLine()`은 한 줄 전체를 읽어옴.
* 사용 후에는 `scan.close()`로 자원을 닫아주는 것이 정석.

#### ③ 형변환 (`CastingTest`)
* **자동 형변환 (Promotion):** 작은 타입에서 큰 타입으로 (예: `byte` → `int`).
* **강제 형변환 (Casting):** 큰 타입에서 작은 타입으로 변환 시 데이터 손실 주의 (예: `(int)doubleVar`).

#### ④ 비교 연산 (`EqualTest`)
* **기본 타입:** `==` 연산자로 값을 직접 비교.
* **참조 타입(String 등):** `==`는 주소값을 비교하므로, 실제 문자열 내용을 비교할 때는 `.equals()` 또는 `.equalsIgnoreCase()`를 사용해야 함.

#### ⑤ 변수의 초기화 및 범위 (`VarInitTest`, `VarWhereTest`)
* **멤버 변수:** 클래스 블록에 선언되며, 초기화하지 않아도 기본값으로 자동 초기화됨.
* **지역 변수:** 메서드 내부에서 선언되며, **사용하기 전에 반드시 초기화**해야 함.
