## 💡 학습 내용

### 1. 자바 실행 흐름 (Execution Flow)
자바는 "Write Once, Run Everywhere"를 실현하기 위해 컴파일과 인터프리트 방식을 혼합하여 사용한다.


1.  **소스 파일 작성:** `.java` 파일 생성
2.  **컴파일 (Compiler):** `javac` 명령어가 소스 파일을 **바이트코드(`.class`)**로 변환
3.  **실행 (JVM):** `java` 명령어로 JVM(자바 가상 머신) 구동
4.  **인터프리트 & 하이브리드:** JVM 내의 **인터프리터**와 **JIT 컴파일러**가 바이트코드를 해당 OS가 이해할 수 있는 기계어로 번역하여 실행

### 2. 자바 에디션 분류
* **SE (Standard):** 표준 에디션. 기본 문법 및 알고리즘용.
* **EE (Enterprise):** 기업용. Servlet/JSP 등 서버 기술 포함. 24시간 운영 환경.
* **ME (Micro):** 임베디드용 (현재 거의 미사용).

### 3. 변수와 자료형 (Data Type & Byte)
변수는 데이터를 담는 그릇이며, **재활용 가능한 메모리 영역**을 의미한다.

| 분류 | 타입 | 크기 | 특징 |
| :--- | :--- | :--- | :--- |
| **정수** | `byte` | 1 byte | -128 ~ 127 |
| | `short` | 2 byte | |
| | `char` | 2 byte | 유니코드 문자 (음수 없음) |
| | **`int`** | **4 byte** | **자바 정수 연산의 기본** |
| | `long` | 8 byte | 큰 정수 (접미사 `L` 추가) |
| **실수** | `float` | 4 byte | 접미사 `f` 추가 |
| | **`double`**| **8 byte** | **자바 실수 연산의 기본** |
| **논리** | `boolean` | 1 byte | `true`, `false` |

* **참고:** 자바는 기본적으로 `unsigned`(부호 없는 정수)를 지원하지 않음.

### 4. 자바 명명 규칙 (Naming Convention)
자바는 가독성을 위해 엄격한 명명 규칙을 권장한다.

| 분류 | 표기법 | 예시 | 특징 |
| :--- | :--- | :--- | :--- |
| **클래스** | **PascalCase** | `MemberService` | 모든 단어의 첫 글자를 대문자로 시작 |
| **메서드** | **camelCase** | `getMemberName()` | 첫 단어 소문자, 이후 단어 대문자 |
| **변수** | **camelCase** | `studentAge` | 메서드와 동일 (의미 있는 이름 권장) |
| **상수** | **UPPER_SNAKE** | `MAX_STOCK` | 모두 대문자, 단어 사이 언더바(`_`) |
| **패키지** | **lowercase** | `com.yun.project` | 소문자만 사용, 마침표(`.`)로 계층 구분 |

### 5. 실습 코드 분석 (Code Examples)
어제와 오늘 실습한 예제 코드들을 통해 자바의 기초 문법을 확인했다.

#### ① 출력 및 서식 (`PrintTestt`)
* `printf()`의 `%s`, `%d`, `%f` 지시자와 `%10.3f` 같은 폭 지정 방법 학습.

#### ② 데이터 입력 (`ScannerTest`)
* `Scanner`를 통한 입력을 배우고, 사용 후 `scan.close()`의 중요성 확인.

#### ③ 형변환 (`CastingTest`)
* 자동 형변환(작은 → 큰)과 강제 형변환(큰 → 작은, 데이터 손실 주의)의 차이.

#### ④ 비교 연산 (`EqualTest`)
* 기본 타입은 `==`, 문자열(String) 내용 비교는 `.equals()` 사용.

#### ⑤ 변수의 범위 (`VarInitTest`, `VarWhereTest`)
* 멤버 변수(자동 초기화)와 지역 변수(사용 전 초기화 필수)의 차이.
