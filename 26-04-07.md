## 💡 학습 내용 (Main Concepts)

### 1. 연산자 핵심 요약 (Section 3.1 ~ 3.12)
* **증감 연산자 (`++`, `--`):** 전치(연산 전 반영)와 후치(연산 후 반영)의 실행 시점 차이가 핵심이다.
    * 예: `c = a++;` (대입 후 증가) / `d = ++b;` (증가 후 대입)
* **정밀한 실수 연산:** `double`의 오차 방지를 위해 정확한 계산은 정수 변환 후 수행하는 것이 안전하다.
* **단락 평가 (Short-Circuit Logic):** `||`, `&&`는 좌항에서 결과가 결정되면 우항을 실행하지 않는다.
    * 예: `if (1 < 2 || (5/0) < 2)` → 좌항이 `true`이므로 우항의 에러(`5/0`)가 발생하지 않음.
    
* **특수 값 처리:** 0으로 나눌 때 발생하는 `Infinity`와 `NaN`은 `Double.isNaN()` 등으로 사전 체크가 필수적이다.
* **삼항 연산자:** `(조건식) ? 참일_때 : 거짓일_때` 형식을 통해 코드를 간결하게 줄일 수 있다.

### 2. Wrapper 클래스 상세 분석 (Deep Dive)
기본 자료형(Primitive type)을 객체(Object)로 다루기 위한 '포장 클래스'이다.

#### ① 주요 특징 및 메서드 활용
* **객체화:** `int` → `Integer`, `double` → `Double` 등 primitive 타입을 객체로 변환하여 다룬다.
* **Autoboxing:** 기본 타입과 객체 타입을 자동으로 상호 변환한다.
* **Parsing (문자열 변환):** 외부 문자열 데이터를 숫자로 바꾸는 실무 빈도 1위 기능이다.

#### ② 실습 코드 예시 (WrapperTest)

// 1. 객체 생성 (valueOf 사용 권장)
Integer iVar = Integer.valueOf(100); 

// 2. 문자열 -> 숫자 변환 (Parsing)
String s = "12345";
int intVar = Integer.parseInt(s);    // 정수 변환
double tmpDouble = Double.parseDouble(s); // 실수 변환

// 3. 객체에서 기본 타입 값 추출 (Unboxing)
Integer iVar2 = Integer.valueOf(200);
byte b = iVar2.byteValue();   // byte형으로 추출
double dou = iVar2.doubleValue(); // double형으로 추출
값 비교 시 주의사항 (중요!)
Wrapper는 객체이므로 ==는 주소를 비교한다. 반드시 **.equals()**를 써야 안전하다.


### ① 데이터 타입별 Wrapper & 크기
| 분류 | 기본 타입 | Wrapper 클래스 | 크기 (Byte) | 특징 |
| :--- | :--- | :--- | :--- | :--- |
| **정수** | `byte` | **Byte** | 1 | -128 ~ 127 |
| | `short` | **Short** | 2 | |
| | `int` | **Integer** | 4 | **자바 정수 연산의 기본** |
| | `long` | **Long** | 8 | 리터럴 뒤에 `L` 필요 |
| **실수** | `float` | **Float** | 4 | 리터럴 뒤에 `f` 필요 |
| | `double` | **Double** | 8 | **자바 실수 연산의 기본** |
| **기타** | `char` | **Character** | 2 | 유니코드 문자 |
| | `boolean` | **Boolean** | 1 | `true` / `false` |

### ② 연산자 우선순위 (실행 순서)
| 순위 | 연산자 종류 | 기호 예시 | 연산 방향 |
| :--- | :--- | :--- | :--- |
| **1** | **증감, 부호, 논리 부정** | `++`, `--`, `+`, `-`, `!` | **오른쪽 → 왼쪽** |
| 2 | 산술 (곱셈/나눗셈) | `*`, `/`, `%` | 왼쪽 → 오른쪽 |
| 3 | 산술 (덧셈/뺄셈) | `+`, `-` | 왼쪽 → 오른쪽 |
| 4 | 비교 및 평등 | `>`, `<`, `==`, `!=` | 왼쪽 → 오른쪽 |
| 5 | 논리 (AND/OR) | `&&`, `||` | 왼쪽 → 오른쪽 |
| 6 | 삼항 | `? :` | 왼쪽 → 오른쪽 |
| **7** | **대입** | `=`, `+=`, `-=` | **오른쪽 → 왼쪽** |

### ③ Math 클래스 주요 메서드
| 메서드 | 기능 | 예시 | 결과 |
| :--- | :--- | :--- | :--- |
| `Math.ceil(x)` | **올림** | `Math.ceil(3.14)` | `4.0` |
| `Math.floor(x)` | **내림** | `Math.floor(3.14)` | `3.0` |
| `Math.round(x)` | **반올림** | `Math.round(3.5)` | `4` |
| `Math.max(a, b)` | **최대값** | `Math.max(10, 20)` | `20` |
| `Math.random()` | **난수** | `(int)(Math.random() * 45) + 1` | **1~45 랜덤 정수** |


## 💭 오늘의 요약 및 느낀 점
- **핵심 요약:** 연산자는 단순 기호가 아니라 메모리 범위와 실행 흐름(단락 평가)을 제어하는 도구임을 이해함.
- **나의 생각:** 이미 아는 내용이라도 기초를 다져서 좋았고, 특히 Wrapper 클래스는 단순 포장을 넘어 자바가 객체지향 언어로서 데이터를 처리하는 핵심 방식임을 깨달았다. `parseInt`와 `equals` 비교는 실수하기 쉬운 부분이니 확실히 암기해야겠다.
