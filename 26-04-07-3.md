## 💡 학습 내용 (Main Concepts)

### 1. Pass by Value (자바의 전달 방식)
* **핵심:** 자바는 언제나 **값(Value)을 복사**해서 전달한다. 
* **참조 타입의 경우:** 객체 자체가 아니라 객체의 **'주소값'**을 복사해서 넘긴다.
* **결과:** 메서드 내부에서 객체의 필드를 바꾸면 원본 객체도 바뀌지만, 매개변수에 새 객체를 대입(`new`)해도 원본 변수는 변하지 않는다.

### 2. 문자열 치환: `replace()` vs `replaceAll()`
* **`replace()`:** 단순 문자열을 찾아 바꾼다. 특수문자(`.`, `*`)도 일반 문자로 취급하여 안전하다. 
* **`replaceAll()`:** **정규 표현식(Regex)**을 사용한다. 특정 패턴(숫자만, 공백만 등)을 바꿀 때 강력하지만, 특수문자 사용 시 주의가 필요하다.

### 3. 메인 메서드 매개변수 (`String[] args`)
* 프로그램 실행 시 외부에서 데이터를 배열 형태로 전달받는다. 
* 전달된 값은 항상 **String(문자열)** 타입이므로, 계산에 쓰려면 `Integer.parseInt()` 같은 파싱 작업이 필수다.

### 4. 열거 타입 (Enum) & 무결성
* **정의:** 요일, 계절 등 한정된 수의 상수를 정의하는 타입이다.
* **장점 (무결성):** 허용되지 않는 값이 들어오는 것을 원천 차단하여 코드의 안전성과 가독성을 높인다.
* **실습:** `Calendar.getInstance()`로 얻은 요일 숫자(1~7)를 `EWeek` 상수로 매핑하여 사용함.

## 📑 [핵심 암기 노트] 실무 포인트

### ① Long 리터럴 접미사 `L`
* 자바 정수 기본형은 `int`(4byte)이다.
* `int` 범위를 초과하는 숫자를 사용할 때는 반드시 뒤에 **`L`**을 붙여 `long` 타입임을 명시해야 컴파일 에러가 나지 않는다.
    * 예: `long val = 3000000000L;`

### ② 참조 타입 메모리 구조 
| 영역 | 저장 내용 | 관리 주체 |
| :--- | :--- | :--- |
| **Stack (스택)** | 기본 타입 변수, 참조 변수(**주소값**) | 메서드 종료 시 자동 소멸 |
| **Heap (힙)** | **실제 객체(Instance)**, 배열 | Garbage Collector(GC) |

## 💭 오늘의 요약 및 느낀 점
* **핵심 요약:** 참조 타입은 주소를 통해 객체를 공유하며, Enum을 통해 데이터 무결성을 확보하는 것이 프로젝트 설계의 핵심임을 이해함.
* **나의 생각:** 단순히 코드를 짜는 것보다 자바가 메모리를 어떻게 쓰는지(Stack/Heap) 아는 것이 디버깅에 큰 도움이 될 것 같다. 특히 Enum의 데이터 보호 능력은 프로젝트에서 적극 활용해야겠다.
