# 📅 TIL: 2026-04-07 (챕터 5. 참조 타입 앞부분)

> **Key Concepts:** `#Stack_Heap`, `#Memory_Allocation`, `#Garbage_Collection`, `#Null_Initialization`, `#Pass_By_Value`

참조 타입은 데이터 자체가 아닌 데이터가 저장된 **메모리 주소**를 참조하는 타입이다. 자바의 효율적인 메모리 관리와 객체 공유의 원리를 학습했다.

---

## 💡 학습 내용 (Main Concepts)

### 1. 자바의 메모리 영역 (Stack vs Heap)
참조 타입을 이해하려면 스택과 힙 영역의 상호작용을 알아야 한다. 
* **Stack (스택):** 메서드 내에서 선언된 변수들이 저장된다. 참조 타입 변수는 실제 데이터가 아닌 힙 영역의 **주소값**을 저장한다.
* **Heap (힙):** 실제 객체(String, 배열 등)가 생성되는 공간이다. 여러 변수가 하나의 힙 객체를 동시에 참조할 수 있다.

### 2. 참조 변수의 초기화와 null
* **초기화:** `String hobby = null;`은 변수는 만들었지만 아직 가리키는 실제 객체가 없음을 의미한다.
* **참조 해제:** 변수에 `null`을 대입하면 기존 객체와의 연결이 끊긴다. 이 상태에서 메서드를 호출하면 `NullPointerException`이 발생하므로 주의해야 한다. 

### 3. 쓰레기 객체 (Garbage) 생성 원리
자바의 `String`은 불변(Immutable) 객체이기 때문에 값이 바뀔 때 메모리에서 특수한 동작이 일어난다. 
* **상황:** `hobby = "여행";` 후 `hobby = "영화";`로 변경 시.
* **과정:** 기존 `"여행"` 객체가 수정되는 것이 아니라, 힙에 새로운 `"영화"` 객체가 생성되고 주소값이 바뀐다.
* **결과:** 주소값을 잃어버린 구 객체(`"여행"`)는 **쓰레기(Garbage)**가 되며, **가비지 컬렉터(GC)**가 이를 감지해 메모리에서 자동 회수한다.

### 4. Pass by Value (자바의 전달 방식)
* 자바는 메서드 호출 시 언제나 **값(Value)을 복사**해서 전달한다. 
* 참조 타입을 넘길 때는 객체의 **'주소값'**이라는 데이터가 복사된다.
* 따라서 메서드 안에서 객체의 내부 필드를 수정하면 원본 객체에 영향을 주지만, 매개변수 자체에 새 객체를 대입(`new`)하는 것은 원본 변수에 영향을 주지 않는다.

## 📑 [핵심 암기 노트] 참조 타입 실무 가이드

| 상황 | 코드 예시 | 메모리 상태 및 결과 |
| :--- | :--- | :--- |
| **선언만 함** | `String s;` | 변수 공간만 생성 (초기화 전 사용 불가) |
| **null 대입** | `s = null;` | 힙 객체와의 연결을 끊음 (GC 유도) |
| **값 변경** | `s = "New";` | 새로운 힙 객체 생성 및 주소값 교체 |
| **문자열 비교** | `s.equals("New")` | 주소가 아닌 **실제 데이터**를 비교해야 함 |
